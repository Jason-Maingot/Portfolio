Jason's Weather App

A Flutter App Experiment Using Real-time data

Overview & Role

 This project is a mobile weather application built using the Dart programming language and the Flutter framework. The primary objective was to deepen my understanding of Flutter, particularly features I was less familiar with—such as streaming live data from the web and managing state across multiple Flutter “pages.”

 I served as the sole developer and designer on this project, overseeing all aspects of development, UI design, and implementation.

Web API Integration & Design Process

 This was my first time integrating a web API to stream live data into a mobile application. I chose to create a weather app inspired by the default weather apps found on most smartphones. I utilized the National Weather Service (NWS) public API, which provides real-time weather data for various U.S. locations near major weather stations.

 While the API offered key data points—like temperature, wind speed, precipitation, and brief condition descriptions—it lacked the detailed insights available in more advanced weather apps. To address this, I designed wireframes that balanced simplicity and usability, taking inspiration from existing weather apps while working within the constraints of Flutter's layout system.

A wireframe sketch used to desing the app

 Integrating the API and displaying the data cleanly proved to be a technical challenge. I leaned heavily on the official Dart and Flutter documentation, as well as community examples, to resolve bugs and ensure the data rendered accurately and efficiently.

Final Deliverable

 Once the core functionality was stable, I focused on visual design and user experience. I selected a blue-dominant color palette, associating it with the sky and common weather visuals. To enhance the interface:

  • I added atmospheric background images
  • Applied rounded corners and transparency to content boxes
  • Adjusted typography and spacing to improve readability and visual balance

 The result was a clean, responsive, and informative weather app with a polished front-end experience.

Next Steps

 This project was primarily a personal learning exercise, but it opened up opportunities for future improvement. Possible next steps include:

  • Consolidating content into a single scrollable view for better UX
  • Leveraging Flutter’s cross-platform capabilities to deploy the app on both Android and iOS, as well as web
  • Expanding the use of APIs to include more detailed weather insights or additional user features
